# Journal Entry

**Intermediate Java - Review Activity**
During a review activity, I wrote a program with two classes: Main.java (demo) and Student.java. Student.java had two attributes (age and gpa), with all the default constructors, getters/setters, and toString method.
In my Demo class, I prompted the user to enter the age and gpas of students they wanted to enter. These values were instantiated as objects in a Students array. I then sorted the new array of students using a bubble
sort method (which called upon the getAge() method from Student.java because we were sorting by age), building off of the bubble sort algorithm I built out last week). Taking the resulting array, I allowed the user to 
search the sorted array for a student with the age that the user would enter using a binary search to zero in on the appropriate Student object within that array. This did well to tie together both basic and intermediate
Java concepts.
